컴포넌트 스캔
    컴포넌트 스캔과 의존관계 자동 주입 시작하기
        지금까지는 @Bean이나 xml <bean> 을 통해서 직접 등록해주었는데 수십 수백개가 된다면 어떡해야 할까??

        설정 정보가 없어도 자동으로 스프링빈을 등록하는 컴포넌트 스캔이라는 기능이 해결책!
        의존 관계도 자동으로 주입하는 @Autowired와 함께

        @Component를 붙이면 자동으로 스프링 빈으로 등록이 된다. 그런데 이러면 의존관계 주입을 해주지 못한다 이래서
        @Autowired를 사용하는거야

    탐색 위치와 기본 스캔 대상
        다 스캔하면 너무 오래걸려서 스캔할 위치를 지정해줄 수 있다. 지정패키지부터 하위패키지로 검색
        지정하지 않으면? @ComponentScan이 붙은 클래스의 패키지의 하위패키지까지 검색

        그래서 요즘에는 프로젝트 시작 루트 위치에 두고 디폴트로 검색하게 둔다.

    필터
        컴포넌트 스캔대상에 추가, 제외가능하다.
        BeanA와 BeanB의 애너테이션이 똑같은데 왜 A는 등록되고 B는 등록이 안된다는거야???


    중복 등록과 충돌
        컴포넌트 스캔에서 같은 빈 이름을 등록하면??
        1. 자동 빈 등록 vs 자동 빈 등록
        2. 수동 빈 등록 vs 자동 빈 등록

        두 가지 경우를 생각해본다.
        1번의 경우에는 에러가 발생한다. 그래서 미연에 방지도 가능하고 이런 경우 거의 x
        2번의 경우가 잦다.
        같은 이름인데도 성공 왜? 수동 빈 등록이 우선권을 가진다! 수동 빈이 자동빈을 오버라이딩한다.
        로그에도 오버라이딩했다는것을 알려준다.

        하지만 개발자의 의도와 다른경우가 많다. 그래서 이제 에러가발생한다.

        어설픈 추상화보다는 명확한것이 대부분 낫다. 어설프게 코드수를 줄이려고 하지말자.
